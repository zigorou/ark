# ADR-0004: Auto Commit/Push Behavior for `ark set` and `ark edit`

| | |
|---|---|
| **Date** | 2026-03-20 |
| **Status** | ✅ Accepted |
| **Issue** | closes #3 |

## Context

`ark set` and `ark edit` modify encrypted vault files. After writing, the changes
exist on disk but are not committed to git. The question is how much of the
git workflow ark should automate.

## Discussion

Four options were considered:

| Option | Commit | Push | Notes |
|--------|--------|------|-------|
| Always auto commit+push | ✅ auto | ✅ auto | Breaks on remote-less vaults; no batching |
| **Auto commit, manual push** | ✅ auto | ❌ manual | Balances convenience and control |
| Manual both | ❌ manual | ❌ manual | Reduces ark's value over raw `sops` |
| Configurable | ✅ auto | configurable | Flexible but adds config complexity |

> **For** auto commit, manual push
> - Commit granularity is natural: one write = one commit
> - Push timing stays under user control (avoids triggering webhooks/CI unexpectedly)
> - Works correctly on local-only vaults (no remote)

> **Against** always auto push
> - Remote-less vaults would fail on every write
> - Multiple `ark set` calls cannot be batched into one commit

**Conclusion**: auto commit always; push defaults to manual, opt-in via config.

### Commit message format

Commit messages are generated by ark to be consistent and scriptable:

```
ark: set ark://obsidian/work/api_key
ark: edit obsidian
```

### Remote-less vaults

If no `origin` remote is configured, push is silently skipped — not an error.
This is consistent with the concept that remote is strongly recommended but not required.

## Decision

- [x] `ark set` and `ark edit` automatically run `git commit` after every write
- [x] Commit message format: `ark: set <uri>` / `ark: edit <category>`
- [x] `git push` is **not** run by default
- [x] `sync.auto_push: true` in `~/.config/ark/config.yaml` enables push after every commit
- [x] If no remote is configured, push is silently skipped

## Consequences

- `internal/vault` (or equivalent) must expose a post-write hook that runs `git commit`
- `git` CLI is used for commit/push (consistent with [ADR-0002](0002-ark-init-design.md))
- `sync.auto_push` is added to the config schema alongside the existing `sync.mode` and `sync.interval`
